🔧 IF STILL NOT ACCESSIBLE:
───────────────────────────────────────────────────────────────────────

STEP 1: Verify server is running on this computer
   ☐ Look for a PowerShell/Command window with Python running
   ☐ If not running, double-click: Start-Server.bat

STEP 2: Test from THIS computer first
   ☐ Open browser on THIS computer
   ☐ Go to: http://192.168.1.2:8000
   ☐ If it works here, firewall is the issue
   ☐ If it doesn't work here, server is not running

STEP 3: Verify firewall rule
   ☐ Open PowerShell as Administrator
   ☐ Run: netsh advfirewall firewall show rule name=all | Select-String "Python"
   ☐ You should see "Python 3.12 - MyISP Tools" or similar

STEP 4: Check Windows Firewall settings manually
   ☐ Press Windows + R
   ☐ Type: firewall.cpl
   ☐ Click "Allow an app through Windows Firewall"
   ☐ Look for "Python" in the list
   ☐ Make sure it's checked for Private and Public networks
   ☐ If not listed, click "Allow another app" and browse to:
      C:\Users\vishnu.ramalingam\AppData\Local\Programs\Python\Python312\python.exe

STEP 5: Alternative - Disable firewall temporarily for testing
   ⚠️  WARNING: Only do this for testing, not for permanent use!
   ☐ Open: Control Panel → Windows Defender Firewall
   ☐ Click "Turn Windows Defender Firewall on or off"
   ☐ Select "Turn off Windows Defender Firewall" for Private network
   ☐ Try accessing from another computer
   ☐ If it works now, the issue is definitely firewall
   ☐ IMPORTANT: Turn firewall back on after testing!

STEP 6: Check if computers are on same network
   ☐ On the other computer, open Command Prompt
   ☐ Run: ping 192.168.1.2
   ☐ If "Request timed out" or "Destination host unreachable":
      - Computers are not on the same network OR
      - Network has isolation enabled
   ☐ Contact your IT administrator about network policies

💡 ALTERNATIVE SOLUTION: SHARED NETWORK FOLDER ACCESS
───────────────────────────────────────────────────────────────────────

If firewall/network issues cannot be resolved, use this method:

1. Right-click MyISP_Tools folder
2. Select Properties → Sharing tab
3. Click "Share" button
4. Add "Everyone" and set permission to "Read"
5. Click "Share" and note the network path (e.g., \\DESKTOP-XYZ\MyISP_Tools)
6. Team members open: \\DESKTOP-XYZ\MyISP_Tools\index.html
7. This opens the files directly from network share (no web server needed)

Note: This method works but URLs will show as file:// paths
