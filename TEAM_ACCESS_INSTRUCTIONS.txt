🔧 IF YOU NEED TO ENABLE FILE SHARING:
───────────────────────────────────────────────────────────────────────

If the path doesn't work, file sharing might not be enabled.
To enable it:

METHOD 1: Using Windows Settings
1. Open Settings (Windows + I)
2. Go to Network & Internet
3. Click "Advanced sharing settings"
4. Under "Private network"
   - Turn ON "Network discovery"
   - Turn ON "File and printer sharing"
5. Click "Save changes"

METHOD 2: Using Control Panel
1. Open Control Panel
2. Go to "Network and Sharing Center"
3. Click "Change advanced sharing settings"
4. Turn on network discovery and file sharing
5. Save changes

METHOD 3: Quick Command (Requires Admin)
- Right-click PowerShell → Run as Administrator
- Type: Start-Service LanmanServer
- Press Enter

🔍 TROUBLESHOOTING:
───────────────────────────────────────────────────────────────────────

❌ "Windows cannot access \\CDC2-L-G04FSGN"

SOLUTION 1: Check if computers can see each other
- Open Command Prompt on team member's computer
- Type: ping CDC2-L-G04FSGN
- If "Reply from...", they can reach your computer
- If "Request timed out", network isolation or firewall issue

SOLUTION 2: Enable File Sharing
- Follow the "Enable File Sharing" instructions above

SOLUTION 3: Check Network Profile  
- Ensure both computers are on "Private" network (not Public)
- Open Settings → Network & Internet → Properties
- Should say "Network profile: Private"

───────────────────────────────────────────────────────────────────────

❌ "Access is denied"

SOLUTION:
- Right-click MyISP_Tools folder
- Properties → Sharing tab
- Click "Share" button
- Add "Everyone" with "Read" permission
- Click "Share"

───────────────────────────────────────────────────────────────────────

❌ "Network path was not found"

SOLUTION:
- Confirm your computer is ON
- Check file sharing is enabled
- Try using IP address instead:
  Find your IP: ipconfig in Command Prompt
  Use: \\192.168.x.x\MyISP_Tools\index.html
