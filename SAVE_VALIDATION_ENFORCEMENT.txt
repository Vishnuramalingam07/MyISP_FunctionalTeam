╔══════════════════════════════════════════════════════════════════════════════╗
║                              WHAT CHANGED                                    ║
╚══════════════════════════════════════════════════════════════════════════════╝

BEFORE:
❌ When saving attendance, if the sheet for a month didn't exist, it would
   automatically create a new sheet with basic format
❌ Users weren't aware they should use "Create Master Sheet" first
❌ Automatically created sheets might not include all team members

AFTER:
✅ When saving attendance, if the sheet for a month doesn't exist, it will
   show an error message and require users to create it first
✅ Users are explicitly directed to use the "Create Master Sheet" button
✅ Ensures all sheets are created with the beautiful colorful format and
   ALL team members included from Team Details.xlsx

╔══════════════════════════════════════════════════════════════════════════════╗
║                              HOW IT WORKS NOW                                ║
╚══════════════════════════════════════════════════════════════════════════════╝

WORKFLOW:

1. USER TRIES TO SAVE ATTENDANCE
   • User selects month/year (e.g., April 2026)
   • User makes changes to attendance data
   • User clicks "💾 Save to Master Excel"

2. SYSTEM CHECKS IF SHEET EXISTS
   • Backend checks Master_Attendance.xlsx
   • Looks for sheet named "Attendance April 2026"
   
3A. IF SHEET DOES NOT EXIST → ERROR
   ❌ Error message displayed:
   
   ┌────────────────────────────────────────────────────────────────┐
   │ ❌ 📄 Sheet not created for April 2026. Please use the        │
   │ "Create Master Sheet" button first.                            │
   │                                                                 │
   │ ⚠️ The master sheet for April 2026 has not been created yet.  │
   │                                                                 │
   │ 👉 Please click the "📄 Create Master Sheet" button first to  │
   │ generate a new sheet with all team members, then try saving    │
   │ again.                                                          │
   └────────────────────────────────────────────────────────────────┘
   
   → User must click "Create Master Sheet" first
   → Then retry saving

3B. IF SHEET EXISTS → SUCCESS
   ✅ Attendance data saved successfully
   ✅ Updated cells with status values
   ✅ Recalculated summary columns
   ✅ Success message with details

╔══════════════════════════════════════════════════════════════════════════════╗
║                              USER WORKFLOW                                   ║
╚══════════════════════════════════════════════════════════════════════════════╝

FOR A NEW MONTH (e.g., April 2026):

STEP 1: CREATE MASTER SHEET
   a. Open attendance tracker
   b. Select month: April
   c. Select year: 2026
   d. Click "📄 Create Master Sheet" button
   e. Confirm the action
   f. ✅ Sheet created with all 175+ team members

STEP 2: ENTER ATTENDANCE DATA
   a. Select a lead from dropdown
   b. Click "🔄 Refresh Data" to load team members
   c. Enter attendance status for each team member
   d. Mark dates (P, SL, VG, etc.)

STEP 3: SAVE CHANGES
   a. Click "💾 Save to Master Excel"
   b. ✅ Data saved successfully
   c. Changes visible to all users

REPEAT STEPS 2-3 AS NEEDED:
   • Save can be done multiple times per month
   • Each save updates only the changed cells
   • No need to recreate the sheet

║                              ERROR SCENARIOS                                 ║
╚══════════════════════════════════════════════════════════════════════════════╝

SCENARIO 1: User tries to save without creating sheet
   Problem:  Sheet "Attendance April 2026" doesn't exist
   Error:    "Sheet not created for April 2026. Please use the 'Create 
             Master Sheet' button first."
   Solution: Click "Create Master Sheet" → Wait for success → Try save again

SCENARIO 2: User creates sheet but doesn't refresh data
   Problem:  Attendance tracker shows "Please select a lead"
   Error:    None, but no data to save
   Solution: Select a lead → Click "Refresh Data" → Enter attendance → Save

SCENARIO 3: User tries to save for wrong month
   Problem:  Looking at April data but trying to save to March sheet
   Error:    Data might go to wrong sheet if March exists
   Solution: Always verify month/year selector matches the data being entered

ISSUE: Error says sheet doesn't exist, but I created it
   → Check if month/year in selector matches the sheet
   → Verify sheet name in Master_Attendance.xlsx
   → Sheet name format: "Attendance [Month] [Year]" (e.g., "Attendance April 2026")

ISSUE: I clicked "Create Master Sheet" but save still fails
   → Refresh the page after creating sheet
   → Check if Master_Attendance.xlsx is not open in Excel (file lock)
   → Verify Flask server is running

ISSUE: Error says sheet exists, but I want to recreate it
   → Open Master_Attendance.xlsx
   → Delete the existing sheet for that month
   → Close the file
   → Use "Create Master Sheet" to recreate it

ISSUE: Can I skip "Create Master Sheet" for urgent saves?
   → No, this is by design to ensure data completeness
   → Creating the sheet takes only 5-10 seconds
   → Better to ensure all team members are included from the start
