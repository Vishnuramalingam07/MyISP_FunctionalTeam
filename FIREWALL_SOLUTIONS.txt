✅ SOLUTION 1: NETWORK SHARE (RECOMMENDED - NO FIREWALL ISSUES!)

This completely bypasses the firewall problem by sharing the folder
directly on the network. No web server, no ports, no firewall issues!

📌 STEP 1: Create the Network Share
   👉 Double-click: Create-Network-Share.bat
   👉 Click "Yes" for Administrator permission
   👉 Wait for "NETWORK SHARE CREATED!" message
   👉 Note the computer name shown

📌 STEP 2: Get Your Computer Name
   Method A: Look at the Create-Network-Share.bat output window
   Method B: Press Windows + Pause/Break key (shows under "Device name")
   Method C: Right-click "This PC" → Properties → look for "Device name"
   
   Example: If computer name is "DESKTOP-ABC123", remember this!

📌 STEP 3: Share This Path With Your Team
   Format: \\COMPUTERNAME\MyISP_Tools\index.html
   Example: \\DESKTOP-ABC123\MyISP_Tools\index.html
   
   Replace COMPUTERNAME with your actual computer name from Step 2!

📌 STEP 4: How Team Members Access It
   1. Open File Explorer (Windows + E)
   2. Click in the address bar at the top
   3. Type: \\COMPUTERNAME\MyISP_Tools\index.html
   4. Press Enter
   5. Website opens in browser automatically!

🔧 WHY IS THE FIREWALL BLOCKING IT?
───────────────────────────────────────────────────────────────────────

Windows Firewall blocks incoming connections by default. We tried to add
rules for Python, but Windows Defender may have additional security 
policies that override these rules.

Common reasons our firewall rules didn't work:
1. Windows Defender Advanced Threat Protection blocking
2. Group Policy from domain/corporate IT overriding local rules
3. Python executable path changed or multiple Python versions
4. Antivirus software with additional firewall
5. Network-level isolation policies
