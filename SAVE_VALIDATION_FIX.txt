PROBLEM:
The previous validation implementation had a potential issue where:
1. When Master_Attendance.xlsx file didn't exist, the code created a blank workbook
2. The validation checked if the sheet existed in the workbook
3. However, the workbook object was modified before the validation check
4. This could lead to unclear error handling in edge cases

USER REQUEST:
"If no sheet is available for the current month it should show the message 
create a sheet in Master excel and it should NOT allow us to save in master 
attendance sheet"

╔══════════════════════════════════════════════════════════════════════════════╗
║                              SOLUTION IMPLEMENTED                            ║
╚══════════════════════════════════════════════════════════════════════════════╝

ENHANCED VALIDATION APPROACH:

1. CHECK FILE EXISTENCE FIRST
   ✓ Before any workbook operations, check if Master_Attendance.xlsx exists
   ✓ Store result in file_exists variable

2. CHECK SHEET EXISTENCE (IF FILE EXISTS)
   ✓ Only load workbook if file exists
   ✓ Check if the specific month sheet exists in the workbook
   ✓ Handle both "Attendance January 2026" and "Attendance_January_2026" formats
   ✓ Store result in sheet_exists variable

3. VALIDATE BEFORE PROCEEDING
   ✓ If file doesn't exist OR sheet doesn't exist → BLOCK SAVE
   ✓ Return detailed error message with specific cause
   ✓ Include require_creation flag for frontend handling
   ✓ Log the blocked save attempt for debugging

4. ONLY PROCEED IF VALIDATION PASSES
   ✓ Workbook is already loaded from validation step
   ✓ Sheet is confirmed to exist
   ✓ Proceed with normal save workflow

FILE: app.py
FUNCTION: save_attendance_to_master_excel()
LINES: ~690-740

BEFORE (Old Logic):
───────────────────────────────────────────────────────────────────────────────
# Load or create workbook
if os.path.exists(MASTER_ATTENDANCE_FILE):
    workbook = load_workbook(MASTER_ATTENDANCE_FILE)
else:
    workbook = Workbook()  # Creates blank workbook

# Remove blank default sheet
if 'Sheet' in workbook.sheetnames:
    workbook.remove(default_sheet)

# Check if sheet exists
if sheet_name not in workbook.sheetnames:
    return error  # Validation happens AFTER workbook creation
───────────────────────────────────────────────────────────────────────────────

AFTER (Enhanced Logic):
───────────────────────────────────────────────────────────────────────────────
# Check file existence
file_exists = os.path.exists(MASTER_ATTENDANCE_FILE)
sheet_exists = False

# Check sheet existence (only if file exists)
if file_exists:
    workbook = load_workbook(MASTER_ATTENDANCE_FILE)
    sheet_exists = (sheet_name in workbook.sheetnames or 
                   sheet_name_legacy in workbook.sheetnames)

# VALIDATION - Block save if file or sheet doesn't exist
if not file_exists or not sheet_exists:
    # Build specific error message
    error_msg = f'Sheet not created for {month_name} {year}'
    if not file_exists:
        error_msg += '. Master_Attendance.xlsx file does not exist'
    elif not sheet_exists:
        error_msg += f'. Sheet "{sheet_name}" not found'
    
    # Log the blocked attempt
    print(f"\n❌ SAVE BLOCKED: {error_msg}")
    print(f"   📋 Required action: Use 'Create Master Sheet' button")
    
    # Return error to frontend
    return jsonify({
        'success': False,
        'error': f'{error_msg}. Please use the "Create Master Sheet" button first.',
        'sheet_name': sheet_name,
        'month': month_name,
        'year': year,
        'require_creation': True,
        'file_exists': file_exists,
        'sheet_exists': sheet_exists
    }), 400

# Validation passed - workbook is already loaded, proceed with save
sheet = workbook[sheet_name]
# ... rest of save logic
───────────────────────────────────────────────────────────────────────────────
